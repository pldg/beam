BEAM

A responsive CSS grid framework based on inline-block

Get Started §

  <!DOCTYPE html >
  <html lang="en">
  <head>
    <meta charset="utf-8">
    <meta http-equiv="X-UA-Compatible" content="IE=edge">
    <meta name="viewport" content="width=device-width, initial-scale=1">
    <title>Beam Grid</title>
    <!--[if lt IE 9]>
      <script src="html5shiv.min.js"></script>
    <![endif]-->
    <link rel="stylesheet" href="beam.min.css">
    <link rel="stylesheet" href="style.css">
    <!--[if lt IE 9]>
      <script src="respond.min.js"></script>
    <![endif]-->
  </head>
  <body>
  </body>
  </html>

Link beam.min.css or beam-lite.min.css in your HTML document's head before your main stylesheet.

Html5shiv polyfill add support of HTML5 elements in IE8. Download html5shiv.min.js and link it before all your stylesheets.

Respond polyfill add support for media queries in IE8. Download respond.min.js and link it after all your stylesheets. Respond.js doesn't work with pages viewed via the file:// protocol (like when opening a local HTML file), to test responsive features in IE8 view your pages over HTTP(S).

Both Html5shiv and Respond polyfills are not necessary for the grid to work.

Grid System §

  • .row > .s*
  • .s* stand for "span" and it's used to set the width of the column
  • Columns must be immediate children of rows
  • You should not place more than 12 columns within a single row (you can have three <div class="s4"> columns in a row)
  • Contents should be placed within columns
  • To avoid layout alignment problems a simple CSS Reset is applied to all elements (read comments in beam.css)

Breakpoints §

Extra Small Small Medium Large Extra Large
Class prefix .xs-* .sm-* .md-* .lg-* .xl-*
Media queries <576px ≥576px
<767px		 ≥768px
<991px		 ≥992px
<1199px		 ≥1200px
.container width 95% 90% 90% 90% 90%

Breakpoint-specific-class has effect on a specific range of window size (.sm-3 class will not affect {xs} {md} {lg} {xl}).

You can set max-width for the .container or even a fixed width if you wish, but is not necessary for the grid to work.

Utility classes §

  • .middle
  • .bottom
  • .center
  • .no-gutters
  • .collapse
  • .delete
  • .hide
  • .first
  • .last
  • .off-*

Utility classes are applied by default to all breakpoints. They can also be applied at specific breakpoints by adding them at the end separated by a hyphen (.delete-lg-xl will only affect {lg} and {xl} but not the other breakpoints).

You can only add one breakpoint per single .off-* class (.off-2-sm, .off-2-md, and so on).

If you use beam-lite.css only .delete has the option for adding breakpoints. The other utility classes are still there but you can only apply them to all breakpoints.

Resize the browser to see how breakpoints affect the layout in the examples below.
